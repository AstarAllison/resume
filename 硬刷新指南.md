# 浏览器硬刷新指南 — 让 GitHub Pages 显示最新版本

## 为什么需要硬刷新？

你的简历网站部署在 `https://astarallison.github.io/resume/`，内容来自 GitHub 仓库 `AstarAllison/resume`。每次更新文件后，GitHub Pages 会重新构建并部署到全球 CDN（Fastly）。但是：

1. **浏览器会缓存 HTML/CSS/JS** — 默认情况下浏览器会保存页面文件几小时甚至几天
2. **CDN 边缘节点有 10 分钟缓存** — 离你最近的 CDN 节点可能还没收到新版本
3. **普通刷新（F5）只会重新请求，不会清缓存** — 所以你看到的是旧版本

硬刷新 = 跳过浏览器缓存，强制从服务器重新拉取。

---

## 各浏览器硬刷新方法

| 浏览器 | Windows / Linux | macOS |
|---|---|---|
| **Chrome / Edge** | `Ctrl + F5` 或 `Ctrl + Shift + R` | `Cmd + Shift + R` |
| **Firefox** | `Ctrl + F5` 或 `Ctrl + Shift + R` | `Cmd + Shift + R` |
| **Safari** | — | `Cmd + Option + R` |

> 如果快捷键没生效，试试下面的「终极方法」

---

## 终极方法 — 打开 DevTools 后刷新

这个方法 100% 有效，会强制跳过所有缓存：

1. 在页面上按 `F12`（macOS 按 `Cmd + Option + I`）打开开发者工具
2. **保持 DevTools 打开状态**（不要关闭）
3. 在地址栏旁边**右键点击刷新按钮**（圆形箭头）
4. 选择「**清空缓存并硬性重新加载**」(Empty Cache and Hard Reload)

Chrome / Edge 截图示意：
```
[ ←  →  C  ⟳ ⋮ ]
             ↑
        右键这个 ⟳
```

---

## 还有问题？直接无痕模式

最简单粗暴的验证方法 — 无痕/隐私窗口里打开：

- Chrome / Edge: `Ctrl + Shift + N`
- Firefox: `Ctrl + Shift + P`
- Safari: `Cmd + Shift + N`

无痕模式**完全不用浏览器缓存**，打开就是服务器最新版。如果无痕模式里看到的是新的，但普通窗口里是旧的，100% 是浏览器缓存问题。

## 排查流程

| 现象 | 原因 | 解决 |
|---|---|---|
| 普通刷新没变化 | 浏览器缓存 | 硬刷新（Ctrl+Shift+R） |
| 硬刷新还没变化 | CDN 边缘缓存 | 等 10 分钟再硬刷新 |
| 等 10 分钟还没变化 | 浏览器缓存很顽固 | DevTools 打开后右键刷新 |
| 上面都不行 | 无痕模式验证 | 仍然旧 → 检查 GitHub Actions 部署日志 |

## 如何确认 GitHub Pages 真的部署了最新版

1. 打开 `https://github.com/AstarAllison/resume/commits/main`
2. 看最上面的提交时间和 commit message
3. 然后打开 `https://astarallison.github.io/resume/`
4. 在页面上右键 → 「查看网页源代码」
5. 搜索你想找的内容（比如 `data-return-section`）— 找到说明部署成功

或者直接命令行验证：

```bash
curl -s https://astarallison.github.io/resume/ | grep -c "data-return-section"
# 应该返回 21
```

## 给手机用户

手机浏览器基本没有"硬刷新"按钮，解决方法：

1. **关闭再打开浏览器**
2. **或者**打开「无痕模式」/「隐私模式」
3. **或者**清除浏览器缓存（设置 → 隐私 → 清除浏览数据）

## 遇到问题时的反馈模板

如果还是看到旧版本，发给我时附上：

```
1. 浏览器和版本（如：Chrome 138）
2. 操作系统
3. 无痕模式下是否正常？
4. DevTools Network 标签里 index.html 的 Status 和 Size
   （Size 显示 "from disk cache" 就是缓存问题）
5. 直接访问 https://raw.githubusercontent.com/AstarAllison/resume/main/index.html
   看到的版本对吗？
```

我看到这些信息就能 1 分钟定位问题。
