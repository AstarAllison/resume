## 一、核心机制：sessionStorage 传递返回路径

**规则：从哪个板块点进来 → 返回按钮就指向哪个板块。**

所有从首页到详情页的跳转，在跳转前把**来源板块 ID** 写入 `sessionStorage.resumeReturnSection`。

详情页加载时，`page-nav.js` 读取这个值：
- 有记录 → 返回按钮显示"返回XX板块"
- 无记录（用户直接访问详情页）→ 使用该页面配置的 `fallbackSection`

**关键原则：没有任何一个详情页的 fallbackSection 是 'home'。**

## 四、page-nav.js 工作流程

```
详情页 <body> 加载完成
    ↓
<script src="js/page-nav.js"></script>
    ↓
调用 initPageNav({ fallbackSection: 'xxx', relatedSections: ['a','b','c'] })
    ↓
读取 sessionStorage.resumeReturnSection
    ├─ 有值 → 用这个值查 SECTION_MAP → "返回工作实习"
    └─ 无值 → 用 fallbackSection → "返回项目经历"
    ↓
生成顶部返回按钮：<a href="index.html#experience">← 返回工作实习</a>
生成底部返回按钮 + 快速导航：
    ← 返回工作实习
    快速前往：[核心能力] [项目经历] [个人作品]
```

## 六、本次修复内容

### 问题 1：项目经历详情页显示"返回首页"
- **原因**：之前部署的是旧版本（没有 page-nav.js），或教育页面的 fallbackSection 设为 'home'
- **修复**：
  - 教育详情页 fallbackSection 从 `'home'` 改为 `'about'`
  - 所有卡片添加显式 `data-return-section` 属性
  - 确保没有任何页面的 fallback 是 `'home'`

### 问题 2：作品集无法点击跳转
- **原因**：`.portfolio-item` 的 CSS 设置了 `cursor: default`
- **修复**：改为 `cursor: pointer`，且所有作品集卡片改用 `data-detail` + `data-return-section="portfolio"` 统一走 `initClickableCards()`

### 问题 3：跳转路径不清晰
- **修复**：更新本文档，列出每一条跳转路径的完整配置
