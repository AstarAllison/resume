# 简历网站需求分析文档

## 项目概述
创建一个专业、有活力的互动式简历网站，给面试官留下深刻印象，展示对待简历的认真程度。

---

## 一、网站架构设计

### 1.1 信息架构（网站地图）

```
简历网站
├── 首页（Home）
│   ├── 个人基本信息展示
│   ├── 短片/轮播展示主要技能
│   └── 工作经历亮点
├── 关于我（About）
│   ├── 个人简介
│   ├── 教育背景
│   └── 联系方式
├── 专业技能（Skills）
│   ├── 设计技能
│   ├── 开发技能
│   └── 其他技能
├── 校园经历（Campus）
│   ├── 主要展示图（可左右滑动）
│   ├── 学生会经历
│   ├── 社团活动
│   └── 其他校园活动
├── 项目经历（Projects）
│   ├── 主要展示图（可左右滑动）
│   ├── 项目卡片网格
│   └── 项目详情页（点击进入）
├── 志愿服务（Volunteer）
│   ├── 主要展示图（照片拼图）
│   ├── 志愿活动卡片
│   └── 活动详情页（点击进入）
├── 实习经历（Internship）
│   ├── 主要展示图（可左右滑动）
│   ├── 时间线展示
│   └── 实习详情页（点击进入）
├── 学校奖项（Awards）
│   ├── 主要展示图（可左右滑动）
│   ├── 奖项卡片网格
│   └── 奖项详情（点击查看）
└── 作品集（Portfolio）
    ├── 主要展示图（可左右滑动）
    ├── 作品筛选功能
    └── 作品详情弹窗/页面
```

### 1.2 页面层级结构

**一级页面（主导航）：** 9个主要页面
- 首页、关于我、专业技能、校园经历、项目经历、志愿服务、实习经历、学校奖项、作品集

**二级页面（详情页）：** 每个一级页面下的具体项目详情
- 点击卡片/项目 → 进入详情页
- 详情页可返回上一级

## 二、设计要求

### 2.1 配色方案

**主色调（淡蓝绿粉紫）：**
- 淡蓝色：`#667eea` (Primary Blue)
- 淡紫色：`#764ba2` (Primary Purple)
- 淡粉色：`#f093fb` (Primary Pink)
- 淡绿色：`#4facfe` (Primary Green)

**渐变色组合：**
- 蓝紫渐变：`linear-gradient(135deg, #667eea 0%, #764ba2 100%)`
- 粉红渐变：`linear-gradient(135deg, #f093fb 0%, #f5576c 100%)`
- 蓝青渐变：`linear-gradient(135deg, #4facfe 0%, #00f2fe 100%)`
- 粉彩渐变：`linear-gradient(135deg, #a8edea 0%, #fed6e3 100%)`

**中性色：**
- 白色：`#ffffff`
- 浅灰：`#f8f9fa`
- 中灰：`#e9ecef`
- 深灰：`#495057`
- 黑色：`#212529`

**设计原则：**
- 活力：使用渐变色和明亮色彩
- 专业：保持色彩协调，不过度饱和
- 细致：注意色彩层次和对比度

### 2.2 排版设计

**字体系统：**
- 主字体：`'Segoe UI', 'Microsoft YaHei', sans-serif`
- 标题字重：Bold (700)
- 正文字重：Regular (400)
- 强调字重：Semi-bold (600)

**字号层级：**
- H1（首页标题）：3rem (48px)
- H2（章节标题）：2.5rem (40px)
- H3（卡片标题）：1.5rem (24px)
- H4（项目标题）：1.25rem (20px)
- Body（正文）：1rem (16px)
- Small（辅助文字）：0.875rem (14px)

**行高：**
- 标题：1.2 - 1.4
- 正文：1.6 - 1.8

### 2.3 间距系统

**基础单位：** 0.25rem (4px)

**间距层级：**
- xs：0.25rem (4px)
- sm：0.5rem (8px)
- md：1rem (16px)
- lg：1.5rem (24px)
- xl：2rem (32px)
- 2xl：3rem (48px)
- 3xl：4rem (64px)

### 2.4 圆角系统

- 小圆角：0.25rem (4px) - 标签、小按钮
- 中圆角：0.5rem (8px) - 卡片、输入框
- 大圆角：1rem (16px) - 大卡片、图片容器
- 超大圆角：1.5rem (24px) - Hero区域、展示区
- 全圆角：50% - 头像、图标容器

### 2.5 阴影系统

- 小阴影：`0 1px 3px rgba(0, 0, 0, 0.12)` - 分隔线
- 中阴影：`0 4px 6px rgba(0, 0, 0, 0.1)` - 卡片
- 大阴影：`0 10px 25px rgba(0, 0, 0, 0.15)` - 悬浮卡片
- 超大阴影：`0 20px 40px rgba(0, 0, 0, 0.2)` - 模态框、下拉菜单

## 三、交互设计要求

### 3.1 页面切换

**要求：**
- 丝滑过渡动画
- 不要过多的页面刷新
- 单页应用（SPA）体验

**实现方案：**
- 使用CSS Transition/Animation
-  opacity + transform 组合
- 持续时间：0.3s - 0.5s

### 3.2 主要展示图交互

**要求：**
- 每个页面都有一个主要展示图区域
- 可以左右滑动切换
- 配上有设计感的元素

**实现方案：**
- 使用轮播/滑块组件
- 支持触摸滑动（移动端）
- 自动播放 + 手动控制
- 指示器（dots）显示当前位置

**特殊页面：**
- 志愿服务页面：照片拼图（Collage）布局
- 使用CSS Grid创建2x2或3x3拼图
- 每张照片可点击放大查看

### 3.3 下拉分级导航

**要求：**
- 在网页左边或右边显示
- 可以随时调整浏览位置
- 层级清晰

**实现方案：**
- 左侧：固定侧边栏导航（桌面端）
- 右侧：下拉菜单/浮动按钮（移动端）
- 当前位置高亮显示
- 点击快速跳转

### 3.4 卡片交互

**要求：**
- 不用过多下滑就能看到大部分信息
- 想要细挖也有足够空间
- 层层点击进入详情

**实现方案：**
- 卡片网格布局（Grid/ Flexbox）
- 卡片悬浮效果（hover）
- 点击卡片 → 进入详情页
- 详情页有返回按钮

**动画效果：**
- 卡片进入视口时淡入
- 每张卡片延迟0.1s，创建交错效果
- 使用Intersection Observer API触发

### 3.5 滚动体验

**要求：**
- 平滑滚动
- 不要突兀的跳转

**实现方案：**
- `scroll-behavior: smooth`
- 使用JavaScript实现自定义滚动动画
- 锚点链接平滑滚动

## 五、技术实现要求

### 5.1 前端技术栈

**核心：**
- HTML5
- CSS3（Flexbox、Grid、Custom Properties）
- JavaScript（ES6+）

**可选增强：**
- 不使用框架，纯原生实现（更轻量）
- 或者使用Vue.js/React（更易维护）

**构建工具：**
- 纯静态，无需构建工具
- 或者使用Vite、Webpack（如果需要框架）

### 5.2 响应式设计

**断点：**
- 桌面端：> 1024px
- 平板端：768px - 1024px
- 移动端：< 768px

**适配策略：**
- 移动端：侧边栏隐藏，使用汉堡菜单
- 平板端：卡片网格调整为2列
- 桌面端：卡片网格3-4列

### 5.3 性能优化

**图片优化：**
- 使用WebP格式（降级到JPEG）
- 懒加载（Lazy Load）
- 图片压缩

**代码优化：**
- CSS/JS压缩
- 减少HTTP请求
- 使用CDN

**交互优化：**
- 防抖（Debounce）和节流（Throttle）
- 使用Intersection Observer API
- 避免强制同步布局

### 5.4 浏览器兼容性

**目标浏览器：**
- Chrome（最新版本）
- Firefox（最新版本）
- Safari（最新版本）
- Edge（最新版本）

**降级方案：**
- 使用CSS @supports检测特性支持
- 提供降级样式
