## 返回逻辑说明

### 当前实现

所有详情页的返回按钮都会读取 `sessionStorage.resumeReturnSection`，然后跳转回 `index.html#${returnSection}`。

### 跳转路径示例

```
用户在首页点击 "核心能力" → 点击 "审计实务(德勤实习)"
→ sessionStorage.resumeReturnSection = "skills"
→ 跳转到 experience-detail.html
→ 点击返回按钮
→ 读取 resumeReturnSection = "skills"
→ 跳转回 index.html#skills ✅

用户在首页点击 "工作实习" → 点击 "德勤中国上海所"
→ sessionStorage.resumeReturnSection = "experience"
→ 跳转到 experience-detail.html
→ 点击返回按钮
→ 读取 resumeReturnSection = "experience"
→ 跳转回 index.html#experience ✅
```

### 已知限制

从 `skill-detail.html` 的"关联经历页面"链接跳转到其他详情页后，返回按钮会回到首页，而不是 `skill-detail.html`。

**解决方案**：如果 HR 从核心能力进入，看完德勤实习详情后，返回按钮会正确回到核心能力板块。如果 HR 从德勤实习详情页点击"关联经历"跳转到其他页面，返回时会回到首页（而不是德勤实习详情页）。

这个限制影响不大，因为大多数 HR 的浏览路径是：首页 → 某个详情页 → 返回首页。

## 版本说明

网站支持 3 个版本，通过右上角按钮切换：
1. **通用版** — 适合大多数岗位（运营、产品、数据分析、财务等）
2. **游戏行业版** — 突出游戏策划/运营相关经历
3. **金融专业版** — 突出财务/审计/金融分析相关经历

切换版本会：
- 更新首页副标题
- 显示/隐藏"游戏分析"能力类别
- 更新简历预览区的经历描述
